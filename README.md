# CAD

A highly interactive Flutter application featuring a fast-paced cartoon slot machine guessing game with real-time global leaderboards.

## Overview

CAD is a Flutter-based mobile application where players race against a 1-minute timer to correctly identify cartoon characters constructed from randomized head, torso, and leg segments generated by a virtual slot machine. Points are awarded for correct guesses, with bonus points for identifying a perfectly matched character. High scores are persisted globally and displayed on a premium, glassmorphic leaderboard.

## Features

- **Slot Machine Mechanic**: Randomized generation of character variations using distinct head, torso, and leg assets from popular cartoons.
- **Timed Gameplay**: A strict 60-second timer to induce fast-paced guessing.
- **Forgiving Input System**: Case-insensitive and whitespace-tolerant text field inputs for seamless user experience.
- **Real-time Global Leaderboard**: Integration with Firebase Firestore to persist and rank player high scores globally.
- **Premium User Interface**: 
  - Custom animated splash screen with a vector graphics (SVG) central logo and shooting star effects.
  - Glassmorphic design language, particularly utilized in the leaderboard screen for a modern aesthetic.
  - A dynamic top-3 podium system for the highest-ranking players.
- **Fluid Animations**: Staggered entry animations, scale transitions, and synchronized slot machine reel physics.

## Tech Stack

- **Framework**: Flutter (Dart)
- **Backend & Database**: Firebase Core, Cloud Firestore
- **Assets & UI Extensions**: `flutter_svg` for vector graphics rendering
- **Tooling**: `flutter_launcher_icons` for native app icon generation

## Project Structure

- `lib/main.dart`: Application entry point and routing configuration.
- `lib/screens/splash_screen.dart`: Animated initialization screen.
- `lib/screens/home_screen.dart`: Main landing and navigation hub.
- `lib/screens/slot_machine_screen.dart`: Core gameplay loop, timer management, and slot machine physics.
- `lib/screens/leaderboard_screen.dart`: Firestore data retrieval and glassmorphic ranking display.
- `assets/`: Directory containing application visuals, including the primary `hero.svg` logo, background images, and structured folders for character parts (`heads/`, `torso/`, `legs/`).

## Setup and Installation

### Prerequisites

- Flutter SDK (version ^3.11.0 or later)
- Dart SDK
- Android Studio or Xcode (for emulation/deployment)
- A configured Firebase project with Firestore enabled.

### Steps

1. Clone the repository to your local machine.

2. Install project dependencies:
   ```bash
   flutter pub get
   ```

3. Configure Firebase:
   - Ensure the `google-services.json` file is placed in the `android/app/` directory for Android builds.
   - For iOS, ensure the `GoogleService-Info.plist` is correctly configured via Xcode.

4. Run the application on a connected device or emulator:
   ```bash
   flutter run
   ```

## Build Instructions

To generate a release APK for Android deployment:

```bash
flutter build apk --release
```

To update the launcher icons after an asset change:

```bash
flutter pub run flutter_launcher_icons
```
